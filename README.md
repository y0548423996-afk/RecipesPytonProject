# מתכונים ב-10 דקות

## סקירה כללית
פרויקט Full-Stack המיועד למעקב, צפייה וניהול של מתכונים בצורה אינטואיטיבית ואסתטית. הפרויקט מאפשר למשתמשים לגשת למאגר מתכונים, לסנן אותם לפי קטגוריות, להוסיף מתכונים חדשים, לערוך או למחוק מתכונים קיימים, וכן לקבל תשובות חכמות בנושא אפייה ובישול באמצעות אינטגרציה ל-Google Gemini.

הפרויקט נבנה כיישום Web End-to-End שממחיש ידע בפיתוח Frontend, Backend, מסדי נתונים, ויצירת חווית משתמש מלאה — דבר שמקנה לו ערך מקצועי במיוחד עבור הצגת יכולות בתעשייה או בראיון עבודה.

---

## מטרת הפרויקט
המטרה המרכזית של הפרויקט היא ליצור פלטפורמה נוחה למשתמשים לשמור, לנהל ולהתייעץ לגבי מתכונים, תוך שימוש בטכנולוגיות מודרניות המאפשרות חיבור בין ממשק משתמש לשרת ולמסד נתונים.

הפרויקט מייצג פתרון מעשי לנושא יומיומי, ובו ניתן לראות שילוב בין:
- עיצוב ממשק משתמש ידידותי
- ביצוע פעולות CRUD על נתונים
- חיבור בין לקוח לשרת באמצעות API
- אינטגרציה עם שירות AI לענף הקולינרי

### פירוט התפקידים של כל חלק
- client/: ממשק המשתמש, כולל תצוגת מתכונים, טפסים, ניווט ותקשורת עם השרת
- server/api/: נקודות API, לוגיקת השרת והאינטגרציה עם ה-AI
- server/db/: שכבת הגישה למסד הנתונים
- server/models/: מודלים להגדרת הנתונים באמצעות Pydantic
- SQL מסד.sql: קובץ יצירת מסד הנתונים והטבלאות עם נתונים התחלתיים

## פונקציונליות API
השרת מספק מספר נקודות גישה ליצירה, קריאה, עדכון ומחיקה של מתכונים:

- GET /recipes - קבלת כל המתכונים
- GET /recipes/{id} - קבלת מתכון לפי מזהה
- GET /recipes/category/{category_name} - קבלת מתכונים לפי קטגוריה
- POST /recipes - הוספת מתכון חדש
- PUT /recipes/{recipe_id} - עדכון מתכון קיים
- DELETE /recipes/{recipe_id} - מחיקת מתכון
- POST /chat - שליחת שאלה ל-AI וקבלת תשובה

## הוראות הרצה
### 1. יצירת מסד הנתונים
הפעל את קובץ ה-SQL המצורף ב-SQL Server כדי ליצור את מסד הנתונים והטבלאות.

### 2. התקנת התלויות
התקן את הספריות הדרושות באמצעות pip:

```bash
pip install fastapi uvicorn pyodbc requests pydantic
```

### 3. הגדרת מפתח API ל-Gemini
במסלול server/api/secret.txt יש להכניס את מפתח ה-API של Google Gemini.

### 4. הפעלת השרת
```bash
python server/api/main.py
```

השרת יעלה בכתובת:
```text
http://127.0.0.1:8000
```

### 5. פתיחת ממשק המשתמש
פתח את הקובץ client/project.html בדפדפן, או הרץ שרת מקומי מתאים עבור התיקייה client/.
